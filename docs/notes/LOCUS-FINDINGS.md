## 3. Numerical conditioning — (none after a dogfooding fix)

The first run flagged two divisions (`core.py:120` divisor `N`, `io.py:370`
divisor `c`) as possibly ill-conditioned. On review these were **bare-variable
denominators** (`1/N`, `1/c`) — "the divisor could be zero" is trivially true of
every variable and would fire on most divisions, i.e. noise. This run **improved
Locus**: the conditioning check now only flags denominator *expressions* that can
cancel to zero (`x − 1`, `a − b`, `x² − y²`), not bare variables. After the fix
graphgraph produces **0** conditioning findings — the two were false positives.

(The genuinely useful case — `1/(x-1)` where the denominator has a specific pole
at `x=1` — is still flagged.)
