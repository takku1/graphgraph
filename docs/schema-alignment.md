# Schema Alignment: Seamless Integration with Graphify

## 1. The Integration Friction (Why it wasn't "Just let me use Graphify")

Previously, combining `graphify` and `graphgraph` required deep manual inspection of the source code. An indexing run from `graphify` produced a [graph.json](file:///C:/Users/dcarn/aiprojects/locus/graphify-out/graph.json) with a schema that immediately crashed `graphgraph` with `KeyError: 'label'`. 

`graphgraph` expected a strict academic schema, while `graphify` outputs a richer, production-ready schema (which includes concept-level nodes, metadata properties, and alternative key mappings).

To make the integration a seamless **"let me use graphify"** workflow, we must align the Shared IR schemas and ensure `graphgraph` natively supports all output structures generated by `graphify`.

---

## 2. Current Schema Mismatch & Fallback Map

We have implemented robust, zero-configuration fallbacks in the parser ([io.py](file:///C:/Users/dcarn/aiprojects/graphgraph/src/graphgraph/io.py)) to map the two schemas.

### Node Property Alignment

| Target Node Field | Expected Key | Graphify Primary Key | Fallback Chain |
| :--- | :--- | :--- | :--- |
| **Node ID** | `id` | `id` | `item["id"]` (Mandatory) |
| **Label** | `label` | `label` | `item.get("label")` $\to$ `item.get("name")` $\to$ `item["id"]` |
| **Kind** | `kind` | `file_type` | `item.get("kind")` $\to$ `item.get("file_type")` $\to$ `item.get("type")` $\to$ `"unknown"` |
| **Path** | `path` | `source_file` | `item.get("path")` $\to$ `item.get("source_file")` $\to$ `""` |
| **Summary** | `summary` | `properties.description` | `item.get("summary")` $\to$ `item.get("properties", {}).get("description")` $\to$ `""` |
| **Facts** | `facts` | *(None / metadata)* | `tuple(item.get("facts") or [])` (Handles missing/null values) |

### Edge Property Alignment

| Target Edge Field | Expected Key | Graphify Primary Key | Fallback Chain |
| :--- | :--- | :--- | :--- |
| **Source** | `source` | `source` | `item["source"]` (Mandatory) |
| **Target** | `target` | `target` | `item["target"]` (Mandatory) |
| **Relation Type** | `type` | `relation` | `item.get("type")` $\to$ `item.get("relation")` $\to$ `"dependency"` |
| **Weight** | `weight` | `weight` | `float(item.get("weight") if item.get("weight") is not None else 1.0)` |

## 3. What Needs to Be Updated in Docs & Code

To fully close the gap and solidify this alignment, the following updates are required across the project files:

### Codebase Actions

1. **Standardize a JSON Schema:**
   Add `src/graphgraph/schema/graph.schema.json` to define the unified codebase graph structure, supporting both indexing formats.
2. **Add a CLI Ingestion Adapter (`graphgraph ingest`):**
   Add a CLI utility to auto-normalize third-party graphs:
   ```bash
   graphgraph ingest --input graphify-out/graph.json --output normalized-graph.json
   ```
3. **Upstream Format Settings to `graphify`:**
   Configure `graphify` config/settings to allow exporting files with standard `path` and `kind` keys directly, aligning with `graphgraph` natively.
